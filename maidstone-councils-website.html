Redesigning Maidstone Borough Council's website

Background

Maidstone Borough Council is a Local Government Authority in the South East of England.

The website gets between four to seven million visits a year and saw a redesign in early 2018. However, since it went live there has been no user research to check whether the designs still meet user needs and didn’t meet accessibility standards. This led to this project. A new project was launched to improve the way a user interacts with the website, meets more user needs from a wider range of internal and external users whilst making it WCAG 2.1 AA accessible by 23rd September 2020.

This project is live and can be viewed on the website.

Team make up

The team was made up of a Product Owner, Interaction Designer, Content Designer, User Researcher and a Developer.

My role

I had a hybrid role in this project. I was the lead designer and developer from initial concept through to final delivery. I was also the accessibility lead for the project. I was also responsible for mentoring a junior member of the team, making sure they were getting practical and real world experience of being a designer in large, high profile projects whilst ensuring they were getting the skills needed to develop their career.

The project was run using Agile methodologies. A backlog was developed based on user needs and work was split into sprints lasting two weeks.

Interaction designer

I was responsible for creating low and high fidelity wireframes and prototypes based on user needs. I looked at solving problems that user needs raised by creating low fidelity prototypes using pen and paper to test my hypothesis. This meant involving members of the team to test my hypothesis and designs against these user needs.

I then created various high fidelity prototypes to expand on these designs. I used Adobe XD to create them. These prototypes included creating page templates like the home page and content pages to designing specific elements that a user will interact with, like cards, links and buttons and their different states.

I also created some of these prototypes in code as proof of concept to stakeholders

Developer

Once the designs were signed off at each iteration, I was then responsible for developing them into code using HTML, CSS/SASS and Javascript.

After I had developed a component or template I tested them to make sure they met WCAG 2.1 AA standards.

User research and user needs

The previous redesign focused on external users and their needs. However, user research uncovered new user needs from a wider range of internal users too.

We’ve gathered research through web page feedback, statistics from Google Analytics, heat maps from software like Hotjar, in person observations and user testing.

Examples of user needs for the user

As a user, I won't use a search, as the words I use won't be used by the Council.
As a user, I'll use an A to Z as a last resort, as content should be easy to find.
As a user, I want to click anywhere in a box, so that I don't have to figure out where to click.

Examples of user needs for the organisation

As Councillor, I want our website to be visually appealing, so that I'm happy to refer my constituants to it.
As a Councillor, I want images of Maidstone so that it's clear what website a user is on.
As a member of staff, I want to be able to promote our services or events so that more people attend.

The problem

Accessibility

The new design needs to be WCAG 2.1 compliant. There were a serveral issues I had to tackle.

Colour contrasts

The colour palette used had combinations that didn't have a contrast ratio of 4.5:1 or higher.

I created a colour palette that became AAA compliant. This colour palette allows the team to use colour combinations that meet contrast ratios that fit within our corporate branding.

Pink had been used as the hover colour for links across the site. It was also being used as the focus state for buttons. This had a contrast ratio of 4.31:1 giving it a AA rating for contrast with a white background.

Pink had been used as the hover colour for links across the site. It was also being used as the focus state for buttons. I removed pink from the colour palette and added more colours from our branding. and replaced it with a dark green and dark purple.

I also added two more darker background colours in blue and purple to give a bit more colour to the designs. When a user hovered over an interactive element that used the blue, green or purple, I created a darker shade to help signify the change in state. These colours were all AAA rated.

Heading structures

I worked with our content designer to audit the website content. One of the main themes that can back from this was the incorrect use of headings.

H2 headings were rarely used as almost every content page skipped from a H1 to a H3. This will cause issues for screen readers. This was down to having a non-styled H2 so it was never used.

I completely revamped the design of the headings. Taking GDS and the NHS as inspiration, every header got a larger font size, larger line height, heavier font weight and greater spacing.

Designs

There were several components that conflicted with each other as well as some interactive elements having inconsistent designs leading to a confusing experience for users.

Links

Links had inconsistent hover and focus states across the website which confused some users. Previously, links had a default state of dark blue that turned pink when hovered over. However, there was no focus state.

To overcome this, I defined every link to be dark blue instead of the grey used for the main body copy. They were also bold and underlined. When a user hovered on the link it would remove the underline and change colour to a dark grey. The focus state was heavily inspired by GDS's link focus state.

The rationale behind this design was make sure colour blind users can still detect a change in state if the can't see the colour change as well as making it easier for users with poor site to notice the change in state with the underline being removed on hover whilst having a unique focus state so that another change of state is easier to notice.

Cards

The previous design had links sitting inside a grey card. When a user hovered over the link within the card the became underlined and the background colour remained the same. There was no focus state.

This led users to try and click or tap anywhere inside the box hoping that something would happen when they actually needed to click or tap on the link itself.

My approach to this problem was to allow a user to click anywhere inside the box. Giving each box a focus state made sure that a user knew they selected that box. I also added a unique hover where the background colour turns to a darker grey, the bottom border colour turns to a dark version and the main heading loses its under. I also gave the card focus state to remain accessible so that users know that they’re interacting with the card.

I used Hotjar to see how users were interacting with these cards. Whilst users were still clicking or touching the title within the card, there were also contacts in other areas of the box.

This validates my approach to making the entire box clickable, especially when it comes to mobile and tablet devices.

Different variations of these cards can be found across maidstone.gov.uk.

Promoting services or events

Our senior leadership team wanted a way of promoting our services or events that happen throughout the year on the website. An example of a service that needs promoting includes new consultations and news items.

After a few iterations of the designs, I created ‘The latest in Maidstone’ section. This section solved a few user needs. Councillors wanted more imagery across the site and wanted the website to be visually appealing and our Senior Leadership needed a way of promoting services.

The live design can be viewed on the home page.

This design was inspired by the cards I mentioned above. I wanted to keep the design as clean and simple as possible so that it is in keeping with the rest of the home page.

From a development point of view, I made sure our content designer can easily take down and replace a card with new content if there is something new to promote.

The new A to Z link can be viewed across the website.

What happens next?

We’re learning from the previous design by making sure more user research is planned to make the website continually meet user needs.

I’m leading on the development of our design system for our website and forms to enable new members of the team to create usable, responsive and accessible products. It will also be publicly available for other organisations to use and contribute to.
